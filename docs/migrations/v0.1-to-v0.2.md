# v0.1 to v0.2

Status: draft for the next release.

For projects generated from `v0.1.0`, keep your game files and update the
framework dependency in `Cargo.toml` to the new release tag when it exists:

```toml
game-starter = { git = "https://github.com/P2949/game", tag = "v0.2.0", package = "game-starter" }
```

Then run:

```bash
cargo check
cargo run
```

Known v0.1 compatibility notes:

- `game_starter::prelude::*` remains the beginner import for generated games.
- `game_kit::prelude::*` is compatibility-only; use
  `game_kit::beginner::prelude::*` or `game_kit::advanced::prelude::*` in
  content crates.
- Beginner callbacks receive the beginner `Game` wrapper, not `GameCtx`.
- Beginner method renames should keep the old method for one release with a
  deprecation note, changelog entry, and migration note.
- Data-driven projects remain on `assets/game.ron` `version: 1` until a
  concrete [game.ron v1 to v2](game-ron-v1-to-v2.md) guide says otherwise.
